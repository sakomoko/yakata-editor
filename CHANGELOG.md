# Changelog

このプロジェクトの注目すべき変更はすべてこのファイルに記録されます。
フォーマットは [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) に準拠しています。

## [Unreleased]

### Changed

- 矢印描画をクリック方式（クリックで点追加→ダブルクリック確定）からドラッグ方式（mousedown→ドラッグ→mouseup確定）に変更
  - 確定後すぐにUndo可能、操作がより直感的に
  - 中間ウェイポイントは確定後にセグメントダブルクリックで挿入可能（既存機能）
- トップレベルエンティティ（rooms, freeTexts, freeStrokes, arrows）の受け渡しを個別引数から `EntitySnapshot` オブジェクトパターンに統一
  - `history.ts`, `persistence.ts`, `clipboard.ts` の関数シグネチャを簡素化
  - `ClipboardData` と `StorageData` が `EntitySnapshot` を extends する形に変更
  - 新エンティティ型の追加時に変更が必要な箇所を大幅に削減

### Added

- タッチデバイス対応（iPad + Apple Pencil）
  - Pointer Events APIへ移行し、マウス・タッチ・ペンを統一的に処理
  - ピンチズーム（2本指）でキャンバスのズーム操作
  - 2本指パンでキャンバスのスクロール
  - 長押し（500ms）でコンテキストメニューを表示
  - Apple Pencilはマウスと同等の精密操作が可能
- `npm run describe` の出力に矢印（Arrow）情報を追加。ウェイポイント座標・色・線幅・ラベルを表示
- Redo機能（⌘Shift+Z / Ctrl+Y）で、Undoしすぎた操作をやり直し可能に
  - 新しい操作を行うとRedoスタックは自動的にクリア
  - Redoスタックは独立50件制限で永続化対応
- コピー＆ペースト機能（⌘C/⌘V）
  - 選択中の部屋・FreeText・FreeStrokeをコピーしてカーソル位置に貼り付け
  - 複数オブジェクトの相対位置を保持して複製
  - WallObject・InteriorObjectも含めて完全に複製
- 複製機能（⌘D）で選択中のオブジェクトを1グリッド右下にオフセット複製
- 鏡面コピー（ミラーペースト）機能
  - 右クリック→「左右反転して貼り付け」で水平ミラー
  - 右クリック→「上下反転して貼り付け」で垂直ミラー
  - WallObject side/offset/hinge、InteriorObject direction、Camera angleを正しく反転
  - ポリゴン部屋の頂点もミラー対応
- 部屋の右クリックコンテキストメニューに「コピー」を追加
- 空白エリアの右クリックメニューに「貼り付け」「左右反転して貼り付け」「上下反転して貼り付け」を追加
- 部屋の右クリックコンテキストメニューに「部屋を削除」を追加
  - 従来のキーボード操作（Delete/Backspace）に加え、右クリックメニューからも部屋を削除可能に
- インテリアオブジェクト（階段・マーカー・カメラ）の部屋間ドラッグ移動
  - ドラッグ中はオブジェクトが部屋の外にも自由に移動できる
  - ドロップ先に別の部屋があれば、その部屋にオブジェクトを移し替え
  - ドロップ先に部屋がなければ、元の部屋の範囲内に戻す
- フォントサイズスライダーのリアルタイムプレビュー
  - 部屋名・マーカー・FreeTextのフォントサイズ変更ダイアログで、スライダー操作時にCanvas上のフォントサイズが即時反映される
  - キャンセル時は元のサイズに復元、OK時はUndoスタックに正しく記録
- マーカー（死体・ピン・テキスト）のラベルフォントサイズを変更できる機能
  - ダブルクリックまたは右クリック→「ラベルを変更」でフォントサイズスライダー（4〜80px）が表示される
  - 「リセット」ボタンでオブジェクトサイズからの自動計算に戻せる
  - リロード後もフォントサイズが保持される
- Shiftキーでグリッドスナップを無効化し、サブグリッド（ピクセル単位）での配置が可能
  - 部屋の作成・移動・リサイズ、壁オブジェクトの移動・リサイズ、自由配置テキストの移動・リサイズに対応
  - Shift解除時は自動的にグリッドに復帰
- 四角形部屋の頂点ドラッグ時に他部屋の頂点・辺へのスナップ機能
  - グリッドスナップを無効化し、非整数座標での自由な頂点配置が可能
  - 他の部屋の頂点に近づくとオレンジ色のインジケーターが表示されスナップ
  - 他の部屋の辺に近づくと緑色のインジケーター（十字付き）が表示され辺上にスナップ
  - 頂点スナップが辺スナップより優先される
  - 正多角形建物での台形部屋の内部分割など、斜め壁上の配置に対応

### Fixed

- 開発サーバー同期でAPI経由で作成・更新したプロジェクトがブラウザに反映されない問題を修正（index マージロジック・既存プロジェクト更新取り込み・同期後のエディタ再読み込み）
- 四角形部屋の `calcAutoFontSize` がBBベースで大きすぎるフォントサイズを返す問題を修正（実際の辺の最小長を使用）
- 四角形部屋の斜め辺で壁オブジェクトリサイズカーソルが不正確な方向を表示する問題を修正（辺の角度に基づく方向決定）
- 四角形部屋のインテリアオブジェクトがBBベースクランプにより部屋の外にはみ出す問題を修正（重心方向への押し戻しロジック追加）
- 四角形部屋に対する隣接壁同期（adjacency）が不正な位置にペア開口を作成する問題を修正（四角形部屋を同期対象から除外）

- レイアウトを縦方向flexboxに変更し、ピクセル固定の高さ指定を排除
- エディタ内部のストレージ直接呼び出し（`persistToStorage`/`persistViewport`）をコールバック（`onAutoSave`/`onViewportChange`）に分離し、App.tsxが保存先を制御する設計に変更

- `context-menu-handler.ts` のメニュー構築ロジックを5つのbuild関数に分割し、配置パターンの共通化・型改善を実施

### Added

- 隣接部屋の壁オブジェクト自動同期機能
  - 壁オブジェクト（ドア・窓・開口）を配置すると、隣接する部屋の共有壁に開口を自動作成
  - 壁オブジェクトの移動・リサイズ・削除に連動してペア開口も更新・削除
  - 部屋の移動・リサイズ・削除時に全ペア開口を自動的に再構築
  - 自動生成された開口はドラッグ移動・リサイズ・削除が不可（コンテキストメニューに案内表示）
  - 壁オブジェクトが共有壁の範囲を超える場合は開口の幅を自動クランプ
- 壁オブジェクト（窓・ドア・開口）の伸縮リサイズ機能
  - 開口部の端をドラッグして幅を変更可能（グリッドスナップ対応）
  - 壁境界を超えない制約と他オブジェクトとのオーバーラップ防止
  - 部屋選択時にオレンジ色のひし形リサイズハンドルを表示
  - 端にホバーするとリサイズカーソル（ew-resize/ns-resize）を表示
- 部屋の連結機能

  - 連結済みの部屋を選択 → 右クリック →「連結解除」で解除
  - 連結はUndoや保存/読込に自動対応
  - 連結グループインジケーター: 部屋を選択したとき、同じlinkGroupに属する部屋の右上隅にカラードットを表示（色はグループごとにUUIDから自動決定）
- 壁に開口（Opening）を配置できる機能
  - 壁の右クリックでコンテキストメニューから「開口を配置」が可能
  - 壁線が途切れるだけのシンプルな表現（ドアの弧や窓の二重線なし）
  - ホールと通路の連絡部分などに使用
  - 開口のドラッグ移動・削除・保存/復元に対応
- 部屋の重なり順序（Z-Order）を変更できる機能
