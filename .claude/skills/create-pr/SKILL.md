# PR作成ワークフロー

ユーザーからPR作成を依頼された際に、ブランチ作成からレビュー対応までを一貫して実行する。

## ワークフロー

### Step 1: ブランチの確認・作成

1. `git branch --show-current` で現在のブランチを確認
2. mainブランチにいる場合、変更内容に基づいた適切なブランチ名で新しいブランチを作成
   - 命名規則: `feat/機能名`, `fix/修正内容`, `docs/ドキュメント内容` など
   - `git checkout -b <branch-name>`
3. 既にフィーチャーブランチにいる場合はそのまま使用

### Step 2: 変更のコミット

1. `git status` と `git diff` で未コミットの変更を確認
2. 未コミットの変更がある場合、適切なコミットメッセージでコミット
3. コミットメッセージは変更内容を簡潔に日本語または英語で記述

### Step 3: ドキュメント更新チェック

1. `update-docs` スキルを呼び出し、ドキュメント更新が必要かチェック
2. 更新が必要な場合はスキルに従いドキュメントを更新・コミット

### Step 4: PR作成

1. `git push -u origin <branch-name>` でリモートにプッシュ
2. `gh pr create` でPRを作成
   - タイトル: 70文字以内で簡潔に
   - ボディ: `## Summary` と `## Test plan` を含める
3. 作成されたPR URLをユーザーに共有

### Step 5: レビューサイクル

1. PR作成後、`pr-review-cycle` スキルの内容に従って処理する**バックグラウンドエージェント**を起動する
   - Agent ツールの `run_in_background: true` を使用
   - プロンプトには PR番号、ブランチ名、変更内容の概要を含める
   - エージェントにはレビュー待機・取得・修正・プッシュまでを一任する
2. バックグラウンドで実行中もメイン会話は継続可能
3. エージェント完了通知が来たら、結果をユーザーに報告する
