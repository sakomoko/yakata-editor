# ドキュメント自動更新スキル

機能の追加・変更・修正が完了した際に、関連するドキュメントを更新する。

## 更新対象ファイル

### 1. CHANGELOG.md（ルート）
- [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) 形式で `[Unreleased]` セクションに追記
- カテゴリ: `Added`（新機能）/ `Changed`（変更）/ `Fixed`（修正）/ `Removed`（削除）
- 日本語で簡潔に記述

### 2. 開発者ドキュメント（docs/dev/）
- **architecture.md** — モジュール構成・データフロー・依存関係に変更があった場合に更新
- **data-structures.md** — 型定義（types.ts）に変更があった場合に更新
- 新しいモジュールやサブシステムが追加された場合は、必要に応じてファイルを追加

### 3. ユーザーガイド（docs/user-guide/）
- **getting-started.md** — 基本的な使い方に影響する変更があった場合に更新
- **controls.md** — 操作方法・ショートカット・ツールバーに変更があった場合に更新
- 新しい機能カテゴリが追加された場合は、必要に応じてファイルを追加

## 更新手順

1. 変更内容を把握する（直前の作業内容、git diffなど）
2. CHANGELOG.mdの `[Unreleased]` セクションに変更を追記
3. 変更が影響するドキュメントを特定し、該当ファイルを更新
4. 仕様書（docs/SPEC.md）の内容と矛盾がないか確認

## 更新ルール

- ドキュメントはすべて**日本語**で記述する
- 既存の記述スタイル・フォーマットに合わせる
- コード例やインターフェース定義は実際のソースコード（src/types.tsなど）と一致させる
- ユーザーガイドは技術用語を最小限にし、操作手順を具体的に書く
- 開発者ドキュメントは正確性を重視する
- 変更に関係ないドキュメントは触らない
