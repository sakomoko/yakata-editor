# ドキュメント自動更新スキル

機能の追加・変更・修正が完了した際に、関連するドキュメントを更新する。

## 発動条件

- 機能追加・変更・バグ修正の実装が完了したとき（プロアクティブに発動）
- ユーザーからドキュメントの更新・確認を依頼されたとき
- PR作成前に、未反映のドキュメント更新がないかチェックするとき

## 更新対象ファイル

### 1. CHANGELOG.md（ルート）
- [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) 形式で `[Unreleased]` セクションに追記
- カテゴリ: `Added`（新機能）/ `Changed`（変更）/ `Fixed`（修正）/ `Removed`（削除）
- 日本語で簡潔に記述

### 2. 開発者ドキュメント（docs/dev/）
- **architecture.md** — モジュール構成・データフロー・依存関係に変更があった場合に更新
- **data-structures.md** — 型定義（types.ts）に変更があった場合に更新
- 新しいモジュールやサブシステムが追加された場合は、必要に応じてファイルを追加

### 3. ユーザーガイド（docs/user-guide/）
- **getting-started.md** — 基本的な使い方に影響する変更があった場合に更新
- **controls.md** — 操作方法・ショートカット・ツールバーに変更があった場合に更新
- 新しい機能カテゴリが追加された場合は、必要に応じてファイルを追加

## 更新手順

1. 変更内容を把握する（直前の作業内容、git diffなど）
2. CHANGELOG.mdの `[Unreleased]` セクションに変更を追記
3. 変更が影響するドキュメントを特定し、該当ファイルを更新
4. 仕様書（docs/SPEC.md）の内容と矛盾がないか確認
5. ショートカットキーの同期チェック（後述）を実行

## ショートカットキー同期チェック

キーボード操作やモードに関する変更があった場合、以下の3箇所が同期されているか必ず確認する。

### チェック対象ファイル
1. **`src/editor/keyboard.ts`** — 実装（信頼できる唯一の情報源）
2. **`src/ShortcutHelpDialog.tsx`** — アプリ内ヘルプダイアログの表示リスト
3. **`docs/user-guide/controls.md`** — ユーザーガイドのキーボードショートカット表

### チェック手順
1. `src/editor/keyboard.ts` の全キーバインドを読み取る
2. `src/ShortcutHelpDialog.tsx` の `shortcuts` 配列と突き合わせ、漏れがないか確認
3. `docs/user-guide/controls.md` の「キーボードショートカット」表と突き合わせ、漏れがないか確認
4. 新しいモードが追加された場合、`controls.md` にモード専用セクションがあるか確認（ペイントモード・アローモードのように独立セクションで操作を説明する）
5. ツールバーにボタンが追加された場合、`controls.md` のフッターバー表にも反映されているか確認

### 判断基準
- keyboard.tsに存在するが他2箇所に記載がない → **追加が必要**
- keyboard.tsから削除されたが他2箇所に残っている → **削除が必要**
- キーバインドが変更された → **3箇所すべてを更新**

## 更新ルール

- ドキュメントはすべて**日本語**で記述する
- 既存の記述スタイル・フォーマットに合わせる
- コード例やインターフェース定義は実際のソースコード（src/types.tsなど）と一致させる
- ユーザーガイドは技術用語を最小限にし、操作手順を具体的に書く
- 開発者ドキュメントは正確性を重視する
- 変更に関係ないドキュメントは触らない
