# 間取り図編集スキル

yakata-editor（館エディタ）のプロジェクトデータをプログラム的に編集する。

## 前提条件

- 開発サーバーが起動していること（`npm run dev`）
- ブラウザを開き直す（またはリロードする）と、起動時の双方向同期でサーバー側の変更がlocalStorageに取り込まれる

## 編集方法

### REST API経由（開発サーバー起動中）

```bash
# プロジェクト一覧
curl http://localhost:5555/api/projects

# プロジェクトデータ取得
curl http://localhost:5555/api/projects/<id>

# プロジェクトデータ更新（全体を上書き）
curl -X PUT http://localhost:5555/api/projects/<id> \
  -H 'Content-Type: application/json' \
  -d '{ ... ProjectData ... }'

# 新規プロジェクト作成
curl -X POST http://localhost:5555/api/projects \
  -H 'Content-Type: application/json' \
  -d '{"name": "新しい館"}'
```

### ファイル直接編集

プロジェクトデータは `data/projects/<id>.json` に保存されている。直接JSONを編集し、ブラウザをリロードすれば反映される。

## データ構造リファレンス

### ProjectData（トップレベル）

```json
{
  "rooms": [],
  "freeTexts": [],
  "freeStrokes": [],
  "viewport": { "zoom": 1, "panX": 0, "panY": 0 },
  "history": []
}
```

### Room（部屋）

```json
{
  "id": "crypto.randomUUID()で生成",
  "x": 5, "y": 3, "w": 4, "h": 3,
  "label": "リビング",
  "floor": 1,
  "wallObjects": [],
  "interiorObjects": []
}
```

- `x`, `y`: 左上座標（グリッド単位、整数推奨）
- `w`, `h`: 幅・高さ（グリッド単位、1以上）
- `label`: 部屋名
- `floor`: 階数（省略時は1）

### WallObject（壁オブジェクト）

```json
// 窓
{ "id": "uuid", "type": "window", "side": "n", "offset": 1, "width": 2 }

// ドア
{ "id": "uuid", "type": "door", "side": "e", "offset": 2, "width": 1,
  "swing": "inward", "hinge": "start" }

// 開口部
{ "id": "uuid", "type": "opening", "side": "s", "offset": 0, "width": 3 }
```

- `side`: n=北, e=東, s=南, w=西
- `offset`: 壁の始点からの距離。北壁・南壁なら左端から、東壁・西壁なら上端から
- `width`: オブジェクトの幅
- **制約**: `offset >= 0` かつ `offset + width <= 壁の長さ`
  - 北壁・南壁の長さ = `room.w`
  - 東壁・西壁の長さ = `room.h`

### InteriorObject（インテリア）

```json
// 階段
{ "id": "uuid", "type": "stairs", "stairsType": "straight",
  "direction": "n", "x": 0, "y": 0, "w": 2, "h": 3 }

// マーカー
{ "id": "uuid", "type": "marker", "markerKind": "body",
  "direction": "e", "x": 1, "y": 1, "w": 1, "h": 1, "label": "被害者" }

// 防犯カメラ
{ "id": "uuid", "type": "camera",
  "x": 0, "y": 0, "w": 1, "h": 1,
  "angle": 0, "fovAngle": 0.523, "fovRange": 5,
  "fovColor": "rgba(0,150,255,0.15)",
  "fovStrokeColor": "rgba(0,150,255,0.4)" }
```

- インテリアの `x`, `y` は**部屋内のローカル座標**（部屋の左上からの相対位置）
- `direction`: n=北, e=東, s=南, w=西
- カメラの `angle`: ラジアン（0=東, PI/2=南, PI=西, -PI/2=北）
- カメラの `fovAngle`: 視野の半角（ラジアン、PI/6 = 30°がデフォルト）
- カメラの `fovRange`: 視野到達距離（1〜20グリッド）

### FreeText（フリーテキスト）

```json
{ "id": "uuid", "gx": 0, "gy": -2, "w": 4, "h": 1,
  "label": "1F", "fontSize": 14, "zLayer": "front" }
```

## 編集ワークフロー

1. **まず現状を確認**: `/describe-floor-plan` で構造を把握
2. **データを取得**: API or ファイルから現在のProjectDataを取得
3. **JSONを編集**: 部屋の追加・変更・削除
4. **バリデーション**: `npx tsx src/cli/validate.ts` で整合性チェック
5. **保存**: API PUT or ファイル書き込み
6. **確認**: ブラウザをリロードして結果を目視確認

## 注意事項

- IDは `crypto.randomUUID()` 形式（例: `"a1b2c3d4-e5f6-..."`)。一意であれば任意の文字列でもよい
- 隣接する部屋にドアや窓を置くと、エディタが自動で対面の壁に開口部（`autoGenerated: true`）を同期する。API経由で追加した場合はこの自動同期は走らないので、必要なら両方の部屋に手動で壁オブジェクトを設定する
- `history` 配列はUndo用のスナップショット。API経由の編集では空配列 `[]` でよい
- `viewport` はカメラ位置。編集に影響しないので既存値を維持すればよい
