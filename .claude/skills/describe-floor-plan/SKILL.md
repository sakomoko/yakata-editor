# 間取り図構造の読み取りスキル

yakata-editor（館エディタ）で作成された間取り図プロジェクトの構造を解析し、わかりやすく説明する。

## 使い方

### CLIツールで構造データを取得

```bash
# プロジェクト一覧と最初のプロジェクトの構造を表示
npx tsx src/cli/describe.ts

# 特定のプロジェクトIDを指定
npx tsx src/cli/describe.ts <projectId>

# JSONファイルを直接指定
npx tsx src/cli/describe.ts path/to/project.json
```

### 開発サーバーAPI経由でも取得可能（開発サーバー起動中のみ）

```bash
# プロジェクト一覧
curl http://localhost:5555/api/projects

# 特定プロジェクトのデータ
curl http://localhost:5555/api/projects/<id>
```

## 構造データの読み方

### 座標系
- 座標は**グリッド単位**（1グリッド = 20px）
- `(x, y)` は左上の座標。x が東西（右が東）、y が南北（**下が南**）
- `w×h` は幅×高さ（グリッド単位）

### フロアの表現方法
- `floor` プロパティで階を示す場合もあるが、多くの場合は**キャンバス上で上下に並べて**複数フロアを表現している
- `FreeText` のラベル（「1F」「2F」など）でフロアが識別できる場合がある
- y座標が小さいほど上（通常は上の階）

### 壁オブジェクト
- **side**: n=北壁, e=東壁, s=南壁, w=西壁
- **offset**: 壁の始点（左上基準）からの距離（グリッド単位）
- **width**: オブジェクトの幅（グリッド単位）
- **type**: window=窓, door=ドア, opening=開口部
- ドアの **swing**: inward=内開き, outward=外開き
- ドアの **hinge**: start=始点側ヒンジ, end=終点側ヒンジ

### 隣接関係
- 2つの部屋が壁を共有している場合、隣接関係として表示される
- 開口部（opening）や壁オブジェクトがある壁で隣接していれば、部屋間を行き来できる
- 名前のない部屋はIDで表示される（廊下やホールなどラベル未設定の場合）

### インテリアオブジェクト
- **stairs**: 階段（straight=直線, folding=折り返し）。direction で上り方向を示す
- **marker**: マーカー（body=人体, pin=ピン, text=テキスト）
- **camera**: 防犯カメラ。angle=向き（度）, fovRange=視野到達距離（グリッド）

## 出力時のガイドライン

CLIの出力を得たら、以下の観点で分析・説明する：

1. **建物全体の概要** — 何階建てか、どんな用途の建物か（住宅、館、施設など）
2. **フロアごとの構成** — 各階の部屋配置をASCII図で視覚化すると特にわかりやすい
3. **動線分析** — 廊下・ドア・開口部の配置から、人がどう移動できるかを説明
4. **特記事項** — 防犯カメラの配置、死体マーカーの位置、吹き抜けや階段の接続など、推理・シナリオ上重要なポイント

## バリデーション

構造の整合性を確認するには：

```bash
npx tsx src/cli/validate.ts [projectId or ファイルパス]
```

エラー（壁オブジェクトの範囲外配置など）や警告（部屋の重なりなど）を報告する。
